Install MeetingNotifier

Plug in your ESP32-C3 via USB and click Install. No PlatformIO. No Python. No terminal.

Your browser doesn't support WebSerial. Use Chrome, Edge, or Opera on a desktop computer (not iOS / Android).

1. Pick your calendar source

Same choice as the README. Pick the one your calendar admin allows; if unsure, try Apps Script / helper first.

Apps Script web app, or local helper

For personal Google Calendars, or work calendars where you'll run the local OAuth helper. Wired up over plain HTTP/HTTPS to a JSON endpoint.

iCal secret URL

For work calendars where Google's Settings → Integrate calendar → Secret address in iCal format still appears. Fetches and parses .ics on-device.

My display is blank / I have an integrated ESP32-C3 + TFT board

If you bought a small board with the display baked into the PCB (Univision / "ESP32-C3 com Display Integrado 1.44"), the GPIOs are wired differently than the standalone modules. This probe firmware cycles through candidate pin maps every ~7 seconds and labels each one on the screen. Note which combo number renders readable text, then open an issue with that number — we'll bake the matching pin map into a permanent build variant.

2. Configure WiFi and your calendar URL

After flashing, the device boots into setup mode and creates a WiFi network called MeetingNotifier-Setup.

  1. On your phone, join the MeetingNotifier-Setup network.
  2. A setup page should pop up automatically. If not, open http://192.168.4.1.
  3. Enter your home / office WiFi name and password.
  4. Paste your calendar URL (the Apps Script /exec URL, the Google iCal "secret address", or your local helper's http://<host>:8080/).
  5. Tap Save and reboot. The device reboots, joins your WiFi, and starts polling.
Need to change WiFi or URL later? Hold BOOT and tap RST while powered to factory-reset, then run setup again. (Coming in a future release: a way to re-open setup without erasing.)
